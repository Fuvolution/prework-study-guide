Prework Study Guide

HTML

  • The head element contains information about the webpage.
  • The body element represents the visible content shown to the user.
  • The attribute of lang="en" tells the browser the language for the webpage is English!
  • An HTML attribute defines additional information about an element!
  • The title element defines the title for a webpage!
  • The style element defines the CSS styles associated with a website!
  • The base element defines the base URL for a webpage!
  • The link element connects an external resource to the HTML document!
  • The meta element defines metadata such as the character set, description, keywords, author, and viewport!
  • The script element embeds client-side scripts in the HTML document!
  • The charset element sets out character encoding!
  • The http-equiv element sets an HTTP header for content!
  • The viewport element sets the default view for our webpage!
  • The elements inside the body element are child elements of the body element (the body element is the parent element)
  • In semantic HTML elements, the name of the element describes that element's content and the role that element plays on a webpage.
  • Head, section, and body elements are examples of semantic HTML elements!
  • Div and span are NOT semantic HTML elements!
  • An element refers to the entire element from opening tag to closing tag!
  • A tag refers only to what appears inside the angle brackets!
  • The p element represents a paragraph or block of text!
  • The heading elements (h1, h2, h3, h4, h5, h6) represents the level of heading a given text block represents!
  • The ul element represents unordered lists, which are bulleted lists!
  • The ol element represents ordered lists, which are numbered lists!
  • The li elements represent the list items!
  • The img element contains information about images that are displayed on the webpage!
  • The src attribute defines the location of the image, often the URL of the image!
  • The br element creates a line of empty space, or a line break, between two blocks of content!
  • The alt attribute contains a text string that describes the appearance and functionality of the image!
  • The a element (known as the anchor element) creates links to the same webpage or other webpages!
  • The href attribute points to the URL for the link!
  • Ex: a href="https://nationalzoo.si.edu/animals/giant-panda" surroudned by angle brackets for the opening anchor element!
  • The header element appears at the top of the webpage and contains a navigation menu and/or introductory content!
  • The biggest heading element h1 should only appear once on each webpage!
  • The footer element appears at the bottom of the webpage and usually contains the author, copyright, contact, sitemap, and navigation!
  • Section elements are semantic elements that define a section in a document that contains thematic content (topic titles, etc.)!
  • Id and class attributes are added to the opening semantic HTML elements!

CSS

  • A margin indicates how much space we want around the outside of an element.
  • A padding indicates how much space we want around the content inside an element.
  • In the external CSS file, the period before the name (ex: .card) designates that as a class selector.
  • Any HTML element assigned to the class will gain the property assignments specified from the specificied class selector!
  • We need to link our external CSS stylesheet file to our HTML file to have the properties applied to elements and classes specified in HTML file!
  • To link our HTML and external CSS file together, we use link rel="stylesheet" href="./assets/style.css" surrounded by angle brackets!
  • The rel attribute specifies the relationship between the current document and the linked document resource!
  • The href attribute specifies the location (URL) of the external resource!
  • The href attribute uses a relative path, which means the location is derived from the current location: the index.html file!

Git

  • git status: checks what branch we are currently on
  • git checkout -b branch-name: creates a new branch and switches to it
  • File System Hierarchy: root (/) -> C: -> Users -> Phuc (Home Directory a.k.a ~) -> Directories & Files inside the home directory
  • Git Workflow:
    1. Navigate to the project directory (cd path-to-repository)
    2. Switch to main branch (git checkout main)
    3. Create a new branch (git checkout -b new-branch-name)
    4. Add or make changes to code on VS Code
    5. Stage your changes (git add -A or git add .)
    6. Commit or save your changes on the local repository (git commit -m "Message")
    7. Check that our current branch is up to date with the remote GitHub repository and pull down all updates on the remote repo (git pull origin main)
    8. Push your changes to the remote GitHub repository (git push origin current-branch-name)
    9. On GitHub, select "New pull request"
    10. Select the main branch as the base branch and your current feature branch as the comparison branch
    11. Create the pull request on GitHub
    12. Provide the pull request a title and short description of the changes
    13. Someone merges the pull request (Select "Merge pull reqeust" -> Select "Confirm merge")
    14. Switch to the main branch on the command line/terminal of VS code (git checkout main)
    15. Pull down the updated remote repository's main branch from GitHub to your local machine (git pull origin main)
    16. Delete the old/outdated feature branch (git branch -d old-feature-branch-name)
    17. Repeat Step 3 through Step 17 until the challenge or project requirements are met and you have a minimum viable product!
